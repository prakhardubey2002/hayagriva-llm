Package: hayagriva-llm
Version: 1.2.0

Description:
Structured LLM metadata standard for Node.js packages — llm.package.json and llm.package.txt

Summary:
Hayagriva-llm is a CLI tool that generates structured LLM metadata files (llm.package.json and llm.package.txt) for Node.js packages. It analyzes package manifests and source code to create comprehensive AI-readable descriptions of package functionality, helping AI systems better understand and interact with npm packages. The tool integrates seamlessly with existing package.json workflows and supports TypeScript analysis for accurate type information.

When to use:
Use Hayagriva-llm when you want to make your Node.js package more discoverable and understandable by AI systems. It's particularly valuable for package authors who want their libraries to be properly interpreted by AI assistants, documentation generators, or automated code analysis tools. Choose this package when you need to generate standardized metadata that describes your package's functionality in a structured format that AI systems can parse and utilize.

Reason to use:
- Generates structured LLM metadata automatically
- Improves AI discoverability of your package
- Creates both JSON and text format descriptions
- Integrates with existing package.json workflows
- Supports TypeScript analysis for accurate type information
- Provides CLI interface for easy integration into build processes
- Offers both static and AI-powered analysis modes

Use cases:
- Generating metadata for npm packages to improve AI assistant responses
- Creating standardized package descriptions for automated documentation systems
- Enhancing package searchability in AI-powered code repositories
- Providing structured data for AI code completion tools
- Documenting package functionality for machine learning model training
- Automating metadata generation in CI/CD pipelines
- Creating local observability dashboards for metadata analysis

Side effects:
- reads package.json
- analyzes source code
- writes llm.package.json and llm.package.txt files

Capabilities:
- Analyzes package.json and source code
- Generates llm.package.json and llm.package.txt files
- Supports static TypeScript analysis via ts-morph
- Offers AI-powered analysis using OpenRouter API
- Provides CLI with multiple commands (generate, dashboard)
- Includes local observability dashboard
- Handles both CommonJS and ESM modules
- Supports environment variable configuration
- Provides verbose logging for debugging
- Validates command-line arguments and options

Configuration:
- OPEN_ROUTER_MODEL environment variable
- HAYAGRIVA_LLM_MODEL environment variable
- OPENROUTER_API_KEY environment variable
- Command-line options for mode, API key, model selection
- Port configuration for dashboard server

Integration:
- Compatible with npm package workflows
- Supports CI/CD pipeline integration
- TypeScript project analysis
- Command-line tool for scripting
- Environment variable configuration
- Local dashboard for metadata inspection

Observability:
- Dashboard server at localhost:4177 (configurable)
- Verbose logging option for debugging
- Error reporting with exit codes
- Process lifecycle logging

Security notes:
- Stores OpenRouter API key in environment variables
- Dashboard runs on localhost only
- No network requests except for AI analysis mode
- Reads package files from current working directory
- Writes metadata files to package directory

Limitations / non-goals:
- Requires Node.js 18+
- AI mode requires OpenRouter API key
- Limited to analyzing Node.js packages
- Static analysis may miss dynamic behavior
- Dashboard reads from .hayagriva-llm/ directory only

Examples:
- hayagriva-llm generate --mode static
- hayagriva-llm generate --mode ai --api-key $OPENROUTER_API_KEY
- hayagriva-llm dashboard --port 4177
- hayagriva-llm --help

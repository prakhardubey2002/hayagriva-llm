Package: hayagriva-llm
Version: 5.0.0

Description:
Structured LLM metadata standard for Node.js packages — llm.package.json and llm.package.txt

Summary:
Hayagriva-llm is a Node.js CLI tool that generates structured, AI-readable metadata files (llm.package.json, llm.package.txt) for npm packages to improve their discoverability and interpretability by LLMs, AI assistants, and automated tooling. It analyzes package manifests and source code using static TypeScript analysis or optional OpenRouter-powered AI extraction, and includes utilities for generating coding agent manuals, Cursor rules, local observability dashboards, and AI readiness audits. Designed for package authors, it integrates seamlessly into existing Node.js workflows and supports both CommonJS and ESM module formats.

When to use:
Use Hayagriva-llm when you are a Node.js package author looking to make your package more interpretable and discoverable by LLMs, AI coding assistants, automated documentation tools, and AI-powered code analysis systems, particularly if you want to provide structured, machine-readable context for AI systems interacting with your package. It is ideal for generating standardized metadata files, coding agent operating manuals (AGENT.md), Cursor IDE rules, or assessing your package's AI readiness via automated audit scores, and integrates seamlessly into existing Node.js package workflows, build processes, or CI/CD pipelines. Choose this package if you want to leverage static TypeScript source analysis or optional OpenRouter-powered AI extraction, run a local observability dashboard for metadata inspection, and support both CommonJS and ESM module formats for your package.

Reason to use:
- Automatically generates structured LLM metadata files (llm.package.json, llm.package.txt) for Node.js packages
- Improves package discoverability and interpretability for LLMs, AI assistants, and automated tooling
- Supports static TypeScript analysis via ts-morph and optional AI-powered extraction via OpenRouter
- Generates coding agent operating manuals (AGENT.md) and Cursor IDE rule .mdc files
- Includes AI readiness audit command to generate package AI Readiness Score reports
- Provides local observability dashboard for inspecting generated metadata
- Integrates into build processes and CI/CD pipelines via CLI interface
- Supports both CommonJS and ESM module formats for broad compatibility
- Offers free LLM Router option for ranked free OpenRouter models (no paid API key required)
- Validates inputs and provides verbose debugging logs for troubleshooting

Use cases:
- Generating LLM metadata for npm packages to improve AI assistant response accuracy about package functionality
- Creating standardized package descriptions for automated documentation generators and AI training datasets
- Generating AGENT.md operating manuals for coding agents to correctly use your package
- Producing Cursor IDE rules (.mdc files) to guide AI-powered code completion for your package
- Auditing Node.js packages to generate AI Readiness Score reports for improvement
- Running local observability dashboards to inspect and validate generated LLM metadata
- Automating metadata generation in CI/CD pipelines to keep LLM metadata up to date with package changes
- Enhancing package searchability in AI-powered code repositories and package registries
- Supporting TypeScript projects with accurate static type information in generated metadata
- Using free OpenRouter models via Free LLM Router to generate metadata without paid API keys

Side effects:
- reads process.env variables and .env files via dotenv
- reads package.json and source files from current working directory or specified directory
- analyzes package source code using static or AI-powered methods
- writes llm.package.json, llm.package.txt, AGENT.md, and .cursor/rules/*.mdc files to target directory
- starts local HTTP observability dashboard server on configurable port
- may exit Node.js process with status codes on error

Capabilities:
- Analyzes package.json manifests and source code to extract package functionality
- Generates llm.package.json and llm.package.txt structured LLM metadata files
- Supports static TypeScript analysis via ts-morph for accurate type information
- Offers optional AI-powered metadata extraction using OpenRouter API
- Generates AGENT.md coding agent operating manuals
- Produces Cursor IDE rule .mdc files for AI-powered code completion
- Runs AI readiness audits to generate package AI Readiness Score reports
- Starts local observability dashboard (defaults to localhost:4177) for metadata inspection
- Supports both CommonJS and ESM module formats for broad package compatibility
- Integrates with Free LLM Router for ranked free OpenRouter models
- Provides CLI interface with multiple commands: generate, agent, audit, dashboard
- Supports environment variable and command-line configuration for all options

Configuration:
- OPEN_ROUTER_MODEL environment variable to set default OpenRouter model
- HAYAGRIVA_LLM_MODEL environment variable as fallback model configuration
- OPENROUTER_API_KEY environment variable for AI mode authentication
- FREE_LLM_ROUTER_API_KEY environment variable for Free LLM Router access
- --mode command-line option to set extraction mode (static or ai)
- --api-key command-line option to pass OpenRouter API key
- --model command-line option to specify OpenRouter model
- --include-src command-line option to include full source in AI prompts
- --freellmrouter command-line flag to use Free LLM Router (implies AI mode)
- --rule command-line flag to generate Cursor rule .mdc files
- --port command-line option to set dashboard server port (default 4177)
- --verbose command-line flag to enable debug logging
- --out command-line option to set AGENT.md output filename
- --force command-line flag to overwrite existing AGENT.md files
- --dir command-line option to set audit target directory

Integration:
- Generates Cursor IDE rule .mdc files for AI-powered code completion and context
- Compatible with standard npm package workflows and package.json scripts
- Supports CI/CD pipeline integration via CLI commands
- TypeScript project analysis via ts-morph for accurate type metadata
- AGENT.md files provide operating context for coding agents (e.g., Cursor, GitHub Copilot)
- LLM metadata files improve package indexing for AI-powered code search tools
- Build process automation via CLI integration with bundlers and test runners

Observability:
- Local dashboard server at configurable localhost port (default 4177) for metadata inspection
- Verbose debug logging option for all CLI commands
- Error messages output to stderr with context for troubleshooting
- Process exit codes indicate success (0) or failure (1)
- Audit command generates AI Readiness Score reports with actionable insights

Security notes:
- API keys (OpenRouter, Free LLM Router) should be stored in environment variables or .env files, not hardcoded
- Local dashboard binds to localhost only by default, no external network exposure
- Only makes outbound network requests when using AI mode or Free LLM Router
- Reads files only from the current working directory or specified target directory
- Writes metadata files only to the current working directory or specified target directory
- No execution of untrusted code during analysis

Limitations / non-goals:
- Requires Node.js 18 or higher
- AI mode requires OpenRouter API key (or Free LLM Router API key for free tier)
- Limited to analyzing Node.js packages only
- Static analysis may miss dynamic code behavior or runtime-generated functionality
- Dashboard reads metadata only from .hayagriva-llm/ directory
- Free LLM Router depends on external free model availability from OpenRouter
- CLI may exit process with status codes on errors

Examples:
- hayagriva-llm generate --mode static
- hayagriva-llm generate --mode ai --api-key $OPENROUTER_API_KEY
- hayagriva-llm generate --freellmrouter --rule
- hayagriva-llm agent --out AGENT.md --force
- hayagriva-llm audit --dir ./src
- hayagriva-llm dashboard --port 8080
- hayagriva-llm --help
